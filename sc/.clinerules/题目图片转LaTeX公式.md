# 题目图片转 LaTeX 公式块 Skill

当用户要求将 Markdown 文档中的**手写数学题目图片**转换为 LaTeX 公式块时，按以下规则执行。

---

## 0. 触发条件

用户指令包含以下关键词之一：
- "将图片转换为公式"
- "把图片转为公式"
- "图片转公式"
- "题目图片转为代码公式"
- "将文档中的题目图片转换为代码公式"
- 其他语义类似的指令

---

## 1. 核心流程

### 1.1 扫描文档，定位目标图片

遍历文档中的所有 `![...](...png)` 图片，识别哪些是**手写数学题目/解答**图片：
- **目标图片特征**: 图片内容为手写的数学公式、极限计算、推导过程等
- **跳过图片特征**: 图片内容为 PPT 讲义截图、概念图、表格截图等非手写内容

### 1.2 逐个处理目标图片

对每个目标图片：
1. 使用**视觉能力**查看图片，识别手写数学内容
2. 将手写内容完整转换为 LaTeX 公式
3. 在图片**之前**插入公式块
4. **保留原图片**不删除（作为参考/对照）

---

## 2. 输出格式

### 2.1 基本结构

**转换前**:
```markdown
## 第四题

56min00s

![image-xxx](./path/image-xxx.png)
```

**转换后**:
```markdown
## 第四题

56min00s

$$
\begin{aligned}
& [手写内容的 LaTeX 转写]
\end{aligned}
$$

![image-xxx](./path/image-xxx.png)
```

### 2.2 公式块规则

- 使用 `$$ ... $$` 包裹
- 内部使用 `&` 对齐的 `aligned` 环境
- 每行推导用 `\\` 换行
- **公式块前后必须有空行**（GitHub 渲染兼容）
- 公式块插在图片**之前**，图片**保留不动**

### 2.3 转写规则

| 手写内容 | LaTeX 转写 |
|----------|-----------|
| 极限 | `\lim_{x \to a}` |
| 分数 | `\frac{分子}{分母}` |
| 乘积/点乘 | `\cdot` 或 `\sdot` |
| 等价符号 | `\sim`、`\Rightarrow` |
| 中文文字 | `\text{...}` |
| 箭头推导 | `=>` 保持 `=>`（与文档风格一致） |
| 下标/上标 | `x_1`、`x^2` |
| 根号 | `\sqrt{x}`、`\sqrt[n]{x}` |
| 希腊字母 | `\alpha`、`\beta`、`\pi` 等 |
| 无穷 | `\infty` |

### 2.4 手写识别注意事项

- 手写可能不清晰，尽力识别，如有不确定的地方**标注说明**
- 注意区分相似符号（如 `1` 与 `l`、`0` 与 `o`）
- 检查公式的数学逻辑是否自洽
- 保持推导步骤与原图一致，不要省略步骤

---

## 3. 与文档风格保持一致

- 参考文档中**已有的公式块风格**（如 `\begin{aligned}` 的使用方式、`&` 的对齐位置）
- 参考文档中 `\text{}` 的使用习惯
- 参考文档中步骤描述文字的格式（如中文注释是否放在 `\text{}` 中）

---

## 4. 执行策略

由于需要逐个查看图片并转写，建议**逐个处理**：
1. 先读取文档，列出所有目标图片
2. 逐个查看图片（使用视觉能力）
3. 使用 `replace_in_file` 在每个图片前插入公式块
4. 完成后验证几个关键图片的转写是否正确
