# 协程

## 什么是协程

协程，就是将一个操作进行分片执行，传统的执行路径是在这一帧内，完整执行整个逻辑和计算。或者手动的对内容进行分状态，如打字机的一个字一个字出现的效果。

CoroutineIEnumerator

## 协程的原理

利用编译器生成的`IEnumerator`自动管理局部状态。

核心: 编译器会将`yiled return`转换为一个`IEnumerator`并将其成员提升为字段，每次迭代都会保存成员状态。

```cs
internal class Program
{
    private static void Main(string[] args)
    {
        Console.WriteLine("Hello, World!");
        _ = Enumerator();
    }

    public static IEnumerator Enumerator()
    {
        int count = 5;
        while(count > 0) {
            count --;
            yield return null;
        }
        yield break;
    }
}
```

> 编译后 用ILSpy看

相当于，也就是一个独立类型

```cs
public class IEnumeratorA
{
    public int count = 5;
    public void Enumerator()
    {
        if (count > 0) {
            count--;
        }
    }
}
```

虽然实际上更复杂，但你只要理解了这个，那么实际原理也就清晰明了了



## 实现一个协程管理器

> 了解了协程的基本概念，接下来我们需要实现一个简单的协程管理器。
>
> 既然是一个管理器，那么肯定要实现基础的增，删，执行。

在实现之前，我们还是必须要知道，由编译器生成的`Enumerator`类型，未实现`Reset`，也就是说，当前逻辑到底后，也就是`yiled break;`后，我们是没办法用`Reset`来实现从头开始的。

### 类型的选择

在`tModLoader`中，拥有大量生命周期的类型为`ModSystem`，因此，在实现协程管理器时，我们也可以非常自然的选择使用`ModSystem`作为我们的心脏，实现帧更新。当然，更好的选择是直接使用`On_Main.Update`

```cs
public class 协程 :  ModSystem {}
```



### 增

首先考虑的就是增，有了增，才能从外部增加，从而推动删和执行。增加也不能粗暴的传入`IEnumerator`，传入的迭代器肯定是外部无法干扰的，独立的，可重复的。因为我们无法调用`IEnumerator.Rest()`。

用什么存储呢？既然叫管理器，那么说明有很多协程，使用`List`好像是一个合适的选择？但是如果使用`List`，我们如何为每个协程任务设置唯一标识呢？使用`List<(Guid, 协程)>`吗，这一结构有另一个变体`Dictionary<Guid, 协程>`。

因此，我们可以考虑使用如下类型: `Func<IEnumerator>`，`Dictionary<Guid, 协程>`。

---

同时，我们还需要考虑到协程都有什么特殊功能，如: `等待`，`间隔`。

为此，我们可以设计一个基础类型，此类型大概可以如下

```cs
public class 协程模型
{
    /// <summary>
    /// 唯一标识
    /// </summary>
    public Guid Code { get; set; }
    /// <summary>
    /// 获取<see cref="Run"/>的方法
    /// </summary>
    public Func<IEnumerator> Orig { get; set; }
    /// <summary>
    /// 实际的迭代器
    /// </summary>
    public IEnumerator Run { get; set; }
    /// <summary>
    /// 是否循环
    /// </summary>
    public bool IsLoop { get; set; } = false;
    /// <summary>
    /// 是否是子协程
    /// </summary>
    public bool IsSub { get; set; } = false;
    /// <summary>
    /// 间隔
    /// </summary>
    public TimeSpan IntervalTime { get; set; } = TimeSpan.FromSeconds(1.0f);
    /// <summary>
    /// 已等待时间
    /// </summary>
    public TimeSpan WaitTime { get; set; } = TimeSpan.Zero;
    public bool MoveNext()
    {
        return Run.MoveNext();
    }

    public void Reset()
    {
        Run = Orig.Invoke();
    }

    /// <summary>
    /// 检查已等待时间
    /// </summary>
    public bool CheckWaitTime_If_True_Then_Reset()
    {
        if(IntervalTime.Ticks - WaitTime.Ticks < 0) {
            WaitTime = TimeSpan.Zero;
            return true;
        }
        return false;
    }
}
```

---

除了基础类型，为了使添加协程侧，能合理的取消此协程任务，我们也可以设计一个返回，包含一个取消方法。

```cs
public record class 携程管理

    public void Stop()

    }

    internal Action StopAction { get; set; }
}
```

如此，那我们就可以实现基础的增加协程任务了。

```cs
public class 协程 : TeachModSystem
{
    /// <summary>
    /// 携程池
    /// </summary>
    public readonly static Dictionary<Guid, 协程模型> pool = [];
    public static 携程管理 Add(Func<IEnumerator> enumerator, bool isLoop = true)
    {
        return Add(enumerator, TimeSpan.FromSeconds(0), isLoop);
    }
    public static 携程管理 Add(Func<IEnumerator> enumerator, TimeSpan intervalTime, bool isLoop = true)
    {
        var code = Guid.NewGuid();
        var obj = new 协程模型()
        {
            Code = code,
            Run = enumerator.Invoke(),
            IsLoop = isLoop,
            Orig = enumerator,
            IntervalTime = intervalTime
        };
        pool.Add(code, obj);
        return new(){ Guid = code, StopAction = () => Remove(code) };
    }
}
```



### 执行

我们可以选择在`On_Main.Update`执行我们的协程任务，而无需关系`ModSystem`的钩子究竟在什么时候执行。

```cs
static 协程() {
    On_Main.Update += MainUpdate;
}

private static void MainUpdate(On_Main.orig_Update orig, Main self, GameTime gameTime)
{
    orig.Invoke(self, gameTime);
}
```

挂钩完成后，我们就要思考究竟如何执行我们的协程任务了。这需要我们对`IEnumerator`有一个基本的了解。[document](https://learn.microsoft.com/zh-cn/dotnet/api/system.collections.ienumerator)，从先前我们知道，使用`MoveNext`可以使其走一步，并返回是否还有下一个元素。那我们就可以不断的调用这个方法，直到他被完整调用。

```cs
private static void MainUpdate(On_Main.orig_Update orig, Main self, GameTime gameTime)
{
    var removeKeys = new List<Guid>();
    foreach (var item in pool.Values) {
        item.WaitTime += TimeSpan.FromSeconds(1f / 60f);
        if (item.CheckWaitTime_If_True_Then_Reset_WaitTime() && !item.MoveNext()) {
            if (item.IsLoop) {
                item.Reset(); // 循环
            } else {
                removeKeys.Add(item.Code); // 不循环
            }
        }
    }
    foreach (var key in removeKeys) {
        pool.Remove(key);
    }
    orig.Invoke(self, gameTime);
}
```

是不是很简单？实际上，这样一来，整个协程系统的核心就已经完成了。



### 删

这基本上是整个过程最简单的内容，因为我们只需要从字典中删除协程任务即可，上面的执行中，我们其实已经执行过这一步骤了。

为了解决整个问题，我们可以引入一个新的类型，而不是在调用Remove方法后，直接删除。而是统一删除。虽然直接使用`List<Guid>`也能解决。但这里我还是决定使用`Queue<Guid>`。如果还考虑线程安全的话，可以使用`ConcurrentQueue<Guid>`。

在执行那步，我们每帧都会创建一个新的`List<Guid>`，并且会开辟4个空间，这样会频繁触发垃圾回收，因此我们也可以把他移动到全局当中，避免频繁创建。

这样一来，我们常规的移除和因为任务到期而被移除就可以被合并了。

```cs
private readonly static Queue<Guid> clearQueue = [];
public static void Remove(Guid id)
{
    clearQueue.Enqueue(id);
}

private static void MainUpdate(On_Main.orig_Update orig, Main self, GameTime gameTime)
{
	while (clearQueue.Count > 0) {
    	pool.Remove(clearQueue.Dequeue());
	}
    ...
    orig.Invoke(self, gameTime);
}
```

其实到这里，一个非常基础的协程管理器就完成了，由用户控制的整个流程大概是这样的

| 用户       | 系统                                                         | 返回     |
| ---------- | ------------------------------------------------------------ | -------- |
| 调用Add    | 生成Guid <br>创建协程任务 <br>加入字典<br>构建Remove高阶函数<br>返回协程管理 | 协程管理 |
| 调用Remove | 加入清除队列                                                 | void     |



## 等帧

虽然我们已经完成了协程的基本功能，但现在，主要内容其实是不完善的，因为协程有一个核心的功能是可等待。而在泰拉中，更适合的等待时间是以帧为单位的，因此我称为`等帧`。

实际上这个需求的实现是很简单的，各位可以自己先尝试实现一下。

我们可以不定义类型，让用户直接`yield return int`，但更好的方式是定义一个类型。

```cs
public class AwaitFrames
{
    /// <summary>
    /// 等待帧
    /// </summary>
    public uint Frames { get; set; }
}
```

创建一个`awaitPool`用于标记需要被等待的协程任务，并在执行中，判断用户是否返回了`AwaitFrames`以及此任务是否在`awaitPool`中。

```cs
private readonly static Queue<Guid> clearQueue = [];
private static void MainUpdate(On_Main.orig_Update orig, Main self, GameTime gameTime)
{
    ...
```

## 子协程

一个协程任务中可能返回一个子任务，只有当子任务被执行完毕后再回到主协程中继续执行，对于如何实现，我们可以栈来保存协程任务，而不是单个`协程模型`。

如果`yield return` 了一个新的`IEnumerator`，那么把他压入栈，这样下一次运行的就是新的子协程了。

```cs
public readonly static Dictionary<Guid, Stack<协程模型>> pool = [];
private static void MainUpdate(On_Main.orig_Update orig, Main self, GameTime gameTime)
{
    while (clearQueue.Count > 0) {
        pool.Remove(clearQueue.Dequeue());
    }
    foreach (var stack in pool.Values) {
        var run = stack.Peek();
        #region 等待AwaitFrames
        {
            if (awaitPool.TryGetValue(run.Code, out var awaitF)) {
                awaitF.Frames -= 1;
                if (awaitF.Frames <= 0) {
                    awaitPool.Remove(run.Code);
                }
                continue;
            }
        }
        #endregion

        run.WaitTime += TimeSpan.FromSeconds(1f / 60f);
        if (run.CheckWaitTime_If_True_Then_Reset() && !run.MoveNext()) {
            if (run.IsSub) { // 子直接出栈丢弃
                stack.Pop();
            } else if (run.IsLoop) { // 根 且循环，不出栈
                run.Reset();
            } else {
                clearQueue.Enqueue(run.Code); // 根， 不循环 删除
            }
        }

        #region 处理子协程和AwaitFrames
        if (run.Run.Current != null) {
            if (run.Run.Current is 子协程 sub) {
                stack.Push(new 协程模型()
                {
                    Code = Guid.NewGuid(),
                    IntervalTime = sub.IntervalTime,
                    IsLoop = false,
                    Orig = sub.EnumeratorFactory,
                    Run = sub.EnumeratorFactory.Invoke(),
                    IsSub = true
                });
            } else if(run.Run.Current is AwaitFrames awaitF) { // 等待帧
                awaitPool[run.Code] = awaitF;
            }
        }
        #endregion
    }
}
public struct 子协程
{
    public 子协程()
    {
        IntervalTime = TimeSpan.Zero;
    }

    public 子协程(TimeSpan intervalTime)
    {
        IntervalTime = intervalTime;
    }
    public required Func<IEnumerator> EnumeratorFactory { get; set; }
    public TimeSpan IntervalTime { get; set; }
}
```

因为泰拉有2个线程，一个是绘制线程，一个是更新线程。所以我们可以分为两种协程。此时主要演示绘制协程，其实更新协程是一样的操作。

## 协程等帧

新建一个字典，其类型为
```cs
static Dictionary<GUID, TimeSpan> awaitPool;
```

当`yile return TimeSpan`（或者自定义类型包装`TimeSpan`）时，将当前携程的GUID加入到等待池，当下一次要对此任务进行`Next`时，先检查等待池是否有改任务，如果有 则对`TimeSpan`减 `1 / 60`，当只剩 `0`时候，移出等待池。
