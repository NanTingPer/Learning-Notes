4. 添加数据

值得注意的是，由于数据库存储的是向量，因此除了被向量化的数据外，还需要提供原始数据。因为向量数据是不可读的，仅用于相似度检索。查询时，实际返回的就是`Payload`。

为了使用已经封装好的库，我们需要引入一个Nuget包

```xml
<ItemGroup>
  <PackageReference Include="OpenAI" Version="2.11.0" />
  <PackageReference Include="Qdrant.Client" Version="1.18.1" />
</ItemGroup>
```

```cs
var payload = new Dictionary<string, Value>
{
    { "名字", "克苏鲁之眼" },
    { "血量", "专家200w， 大师600w" },
    { "生成信息", "...." }
};
var point = new PointStruct() 
{ 
    // 因为C# SDK只支持GUID和ulong，直接使用GUID后续数据将难以更新
     Id = Create("克苏鲁之眼"), 
	//向量
     Vectors = await GetEmbedding("克苏鲁之眼是........"),
     Payload = { payload }
};
await qc.UpsertAsync(collectionName: COLLECTION_NAME, points: [point]);

/// <summary>
/// 根据字符串生成确定的 Guid（基于 MD5 哈希）
/// </summary>
static Guid Create(string input)
{
    using var md5 = MD5.Create();
    byte[] hash = md5.ComputeHash(Encoding.UTF8.GetBytes(input));
    return new Guid(hash);
}

static async Task<float[]> GetEmbedding(string value)
{
    var options = new OpenAIClientOptions()
    {
        Endpoint = new Uri("http://localhost:8080/v1/embeddings")
    };

    var client = new EmbeddingClient("Qwen3-Embedding-0.6B", new ApiKeyCredential("null"), options);
    var embedding = await client.GenerateEmbeddingAsync("你好");
    return embedding.Value.ToFloats().ToArray();
}
```

````markdown
你是主Agent你需要严格执行调用子Agent并实时记录哪些内容已被处理保存到当前目录下的`NPC已处理.json`  `Item已处理.json` ，专门负责将单个游戏Wiki页面（Wikitext格式）转换为严格符合指定JSON Schema的结构化数据。你的输出必须是纯净的JSON对象，不要包含任何解释、Markdown标记或前后缀文本。

## 核心规则
1. 根据页面内容自动判断实体类型，设置`type`字段：
   - "boss"：Boss或敌怪页面（包含生命值、伤害、阶段等）
   - "item"：武器、工具、材料等物品页面（包含伤害、使用时间、稀有度等）
   - "recipe"：纯合成配方页面（主要描述如何合成某物品）
   - "npc"：非Boss的友好或中性NPC
   - "other"：无法明确分类的其他页面

   - `{{价值|0|16}}`：转换为 `{"platinum":0,"gold":16,"silver":0,"copper":0}`。
3. 所有列表类信息（如掉落物品、合成材料、阶段细节）必须用数组对象表示。
````
