# B站学习笔记 Markdown 规范化 Skill

当用户要求将 B站视频学习笔记（Obsidian / Typora 等编辑器导出的 Markdown）规范化为 **GitHub 网页在线预览可正确渲染** 的标准 Markdown 格式时，按以下规则执行。

---

## 0. 前置信息

- **根链接**: 用户会提供或上下文中会包含 B站视频的根链接（如 `https://www.bilibili.com/video/BV1husGzwEtZ`），所有链接均基于此构建。
- **图片目录**: 图片通常与 `.md` 文件同级，路径如 `./章节名/image-xxx.png`。
- **数学公式**: 使用 `$$ ... $$` 块级语法，公式中的中文文本使用 `\text{}` 包裹，**不要抽取或拆分公式中的 `\text{}` 内容**。

---

## 1. `$$` 数学公式块前后必须有空行

**问题**: GitHub 的 Markdown 渲染器要求 `$$` 块的前后行必须是空行，否则无法正确渲染为数学公式。

**规则**:
- 每个 `$$` 开始行的**前一行**必须是空行
- 每个 `$$` 结束行的**后一行**必须是空行
- 连续两个 `$$` 块之间保留**一个空行**

**示例**:
```markdown
<!-- ✗ 错误 -->
函数在无穷处极限存在的充要条件:
$$
\lim_{x\to\infty}f(x) = A
$$
$$
\text{若 } \lim_{x\to-\infty}f(x) \neq \lim_{x\to+\infty}f(x)
$$

<!-- ✓ 正确 -->
函数在无穷处极限存在的充要条件:

$$
\lim_{x\to\infty}f(x) = A
$$

$$
\text{若 } \lim_{x\to-\infty}f(x) \neq \lim_{x\to+\infty}f(x)
$$
```

---

## 2. 裸 B站链接 → 链接化标题

**问题**: 部分标题下方有单独一行的 B站 URL（裸链接），应合并到标题中。

**规则**:
- 如果标题（`#`/`##`/`###` 等）的**下一行**或**下方最近的非空行**是一个纯 B站 URL，则将该 URL 作为链接写入标题
- 删除原来的裸链接行
- 仅处理紧邻标题的裸链接，跳过标题后有其他内容的情况

**示例**:
```markdown
<!-- 原文 -->
### *第三题

https://www.bilibili.com/video/BV1husGzwEtZ?t=790.7&p=17

<!-- 修复后 -->
### [*第三题](https://www.bilibili.com/video/BV1husGzwEtZ?t=790.7&p=17)
```

---

## 3. P 数文字 → 标题链接

**问题**: 大标题（`#` 级）下方有独立的 P 数文字（如 `p18`、`P22`），表示该章节内容来源于视频的某个分P。

**规则**:
- 将标题文本作为链接文字，P 数写入 URL 的 `?p=` 参数
- 删除原来的 P 数独立行

**示例**:
```markdown
<!-- 原文 -->
# 无穷大小

p18

<!-- 修复后 -->
# [无穷大小](https://www.bilibili.com/video/BV1husGzwEtZ?p=18)
```

---

## 4. 时间戳文字 → 链接化标题

**问题**: 子标题下方有时间戳文字（如 `10min 38s`、`03min31s`），表示该内容在视频分P中的时间位置。

**规则**:
- 时间戳格式识别: `Xmin Ys`、`XminYs`、`Xmin`、`Xs` 等变体
- 转换为秒数: `t = X * 60 + Y`
- 将时间戳链接写入标题的 URL 中，格式: `?t={秒数}&p={P号}`
- **P 号从当前最近的父级大标题的 `?p=` 参数中获取**
- 删除原来的时间戳独立行
- **注意**: 如果时间戳后面跟有描述性文字（如 `8min 34s 解释等价无穷小`），则保留描述文字，但将时间戳部分链接化到标题中（如果标题存在）

**示例**:
```markdown
<!-- 原文（父标题 # [无穷大小](...p=18)） -->
## 小的性质

10min 38s

<!-- 修复后 -->
## [小的性质](https://www.bilibili.com/video/BV1husGzwEtZ?t=638&p=18)
```

**P 号确定方法**: 从当前子标题向上查找，找到最近的含 `?p=` 参数的 `#` 级标题链接，提取 P 号。

---

## 5. HTML `<img>` → 标准 Markdown 图片

**问题**: 部分图片使用 HTML `<img>` 标签并带有 `style="zoom:50%;"`，GitHub 会过滤 `style` 属性导致显示异常。

**规则**:
- `<img src="..." alt="..." style="zoom:50%;" />` → `![alt](src)`
- 简单的 zoom 缩放可直接丢弃（GitHub 不支持）
- 如果 `src` 是相对路径，保持不变

**示例**:
```markdown
<!-- 原文 -->
<img src="./章节/image.png" alt="image" style="zoom:50%;" />

<!-- 修复后 -->
![image](./章节/image.png)
```

---

## 6. 多余空行清理

**规则**:
- 连续 3 个及以上空行压缩为 1 个空行
- 文件末尾保留 1 个换行符

---

## 7. 其他保留原则

- **数学公式中的 `\text{}`**: 完整保留，不做任何拆分或抽取
- **已有正确链接的标题**: 不做修改
- **图片路径**: 保持原始相对路径不变
- **引用块 `>`** 和 **代码块** `` ``` ``: 内容原样保留
- **答案、提示等注释性文字**: 保留原位

---

## 执行策略

由于规范化涉及全文几乎所有行，**推荐使用 `write_to_file` 一次性重写整个文件**，逐段应用上述 7 条规则。完成后用 `read_file` 验证关键段落。
